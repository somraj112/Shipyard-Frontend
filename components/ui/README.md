# UI Kit

> Design system primitives for the Shipyard platform

This directory contains reusable, atomic UI components that form the foundation of Shipyard's design system. Components here should be:

- **Generic** — Not tied to any specific feature or domain
- **Composable** — Small, focused, and easily combined
- **Accessible** — Follow WAI-ARIA best practices
- **Themed** — Respect the design tokens defined in `globals.css`

## Planned Components

| Component | Status | Description |
|-----------|--------|-------------|
| `Button` | 🔜 Planned | Primary, secondary, ghost variants |
| `Card` | 🔜 Planned | Content container with header/body/footer |
| `Badge` | 🔜 Planned | Status indicators and labels |
| `Input` | 🔜 Planned | Text, search, and form inputs |
| `Avatar` | 🔜 Planned | User profile images with fallback |
| `Skeleton` | 🔜 Planned | Loading state placeholders |
